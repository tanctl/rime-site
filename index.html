Rime

A privacy-hardened Unified Address light client for Zcash. Addressing linkability that exists outside the cryptography.

GitHub

Why Rime Exists

The privacy gap: Memo-fetch metadata still reveals which shielded outputs belong to a wallet to both lightwalletd and passive observers. Fetch-timing and request patterns allow correlation with specific outputs.

ZIP-314 (reserved) recognized these risks and outlined possible defenses. Rime implements a subset of those ideas in a UA light-client design.

Reference ZIP-314 analysis

Privacy modes

Choose the privacy mode that matches your threat and bandwidth constraints.

Oblivious sync (PIR with fixed-interval dummy fetches) Dual-server PIR with constant-rate queries to hide timing patterns.
cargo run -p rime-cli -- sync --sync-mode pir --pir-server https://pir1 --pir-server https://pir2 --pir-dummy-interval 60s
Full-memo privacy Retrieves all memos to prevent revealing which outputs are relevant.
cargo run -p rime-cli -- sync --sync-mode full-memo

Running Rime

# init once
cargo run -p rime-cli -- init --network testnet --birthday-height 0

# normal selective memos
cargo run -p rime-cli -- sync --endpoint https://testnet.lightwalletd.com:9067

# oblivious PIR (fixed cadence)
cargo run -p rime-cli -- sync --sync-mode pir \
  --pir-server https://pir1.example.com/query \
  --pir-server https://pir2.example.com/query \
  --pir-dummy-interval 60s --tor-only

# full-memo download-all
cargo run -p rime-cli -- sync --sync-mode full-memo --no-confirm
